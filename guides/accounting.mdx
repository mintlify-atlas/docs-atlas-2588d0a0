The accounting module in OpenCBS Cloud provides comprehensive financial management capabilities, including chart of accounts management, transaction recording, and financial reporting.

## Chart of Accounts

The chart of accounts is the foundation of your financial system, defining all accounts used to track financial transactions.

### Understanding Account Structure

OpenCBS Cloud uses a hierarchical account structure:

- **Parent Accounts**: High-level groupings (assets, liabilities, equity, income, expenses)
- **Subgroup Accounts**: Mid-level categorization
- **Balance Accounts**: Leaf-level accounts that hold actual balances

### Account Properties

Each account has the following properties:

**Basic Information**
- **Name**: Account name and description
- **Number**: Account code/number (hierarchical)
- **Branch**: Associated branch (if branch-specific)
- **Currency**: Account currency (if multi-currency)

**Account Type**
- **Is Debit**: Normal balance side (debit or credit)
- Determines how transactions increase/decrease the account

**Transaction Permissions**
- **Allowed Transfer From**: Can be source of transfers
- **Allowed Transfer To**: Can be destination of transfers
- **Allowed Cash Deposit**: Can receive cash deposits
- **Allowed Cash Withdrawal**: Can make cash withdrawals
- **Allowed Manual Transaction**: Allows manual journal entries

**Status**
- **Locked**: Prevents new transactions (closed accounts)

## Managing the Chart of Accounts

### Viewing the Chart of Accounts

<Steps>
  <Step title="Navigate to Accounting">
    From the main menu, click **Accounting** → **Chart of Accounts**.
  </Step>

  <Step title="Browse Account Hierarchy">
    The chart displays as an expandable tree:
    - Click the arrow icon to expand parent accounts
    - View child accounts under each parent
    - Hierarchy shows account relationships
    
    <Tip>
    Use the branch filter at the top to view accounts for a specific branch. Leave blank to view all accounts across all branches.
    </Tip>
  </Step>

  <Step title="View Account Details">
    Click on any account to view its properties and current balance. The system displays:
    - Account number and name
    - Account type (debit/credit)
    - Current balance
    - Branch and currency
    - Transaction permissions
    - Parent account
  </Step>
</Steps>

### Creating New Accounts

<Steps>
  <Step title="Select Parent Account">
    Navigate to the parent account under which you want to create the new account. Click the **Add Account** icon next to the parent account.
  </Step>

  <Step title="Enter Account Details">
    Complete the account creation form:
    
    **Account Number**
    - Child number (appended to parent number)
    - Must be unique under the parent
    - Forms hierarchical account code
    
    **Account Name**
    - Descriptive name for the account
    - Should clearly indicate the account purpose
    
    **Branch**
    - Select the branch this account belongs to
    - Required field
    
    **Currency** (Optional)
    - Select currency if account is currency-specific
    - Leave blank for base currency accounts
    
    **Account Type**
    - Select "Debit" or "Credit" for normal balance
    - Determines transaction increase/decrease behavior
    
    <Note>
    The account's full number is automatically generated by combining the parent account number with the child number you provide.
    </Note>
  </Step>

  <Step title="Set Transaction Permissions">
    Configure what transactions are allowed for this account:
    
    - ☑️ **Allow Transfer From**: Enable as source for transfers
    - ☑️ **Allow Transfer To**: Enable as destination for transfers
    - ☑️ **Allow Cash Deposit**: Enable cash deposits
    - ☑️ **Allow Cash Withdrawal**: Enable cash withdrawals
    - ☑️ **Allow Manual Transaction**: Enable manual journal entries
    
    <Tip>
    Balance (leaf) accounts typically have all permissions enabled. Subgroup accounts usually have permissions disabled as they don't hold balances.
    </Tip>
  </Step>

  <Step title="Save Account">
    Click **Save** to create the account. The new account appears in the chart of accounts hierarchy.
  </Step>
</Steps>

<Warning>
**Maker-Checker Approval**: Depending on your institution's configuration, new accounts may require approval through the maker-checker workflow before becoming active.
</Warning>

### Editing Existing Accounts

To modify an existing account:

1. Navigate to the account in the chart of accounts
2. Click the **Edit** icon next to the account
3. Modify allowed fields:
   - Account name
   - Transaction permissions
   - Locked status
4. Click **Save**

<Note>
Account numbers and parent relationships cannot be changed after creation. To restructure accounts, create new accounts and transfer balances.
</Note>

### Account Organization Best Practices

<Tip>
**Chart of Accounts Tips:**

**Numbering Conventions**
- Use consistent numbering schemes (e.g., 1000-1999 for assets)
- Leave gaps in numbering for future accounts
- Use meaningful prefixes for easy identification

**Naming Standards**
- Use clear, descriptive names
- Follow consistent naming patterns
- Include account type in name if helpful

**Structure**
- Group similar accounts together
- Keep hierarchy depth manageable (3-4 levels typical)
- Create accounts based on reporting needs
</Tip>

## Journal Entries

Journal entries record financial transactions in the accounting system. OpenCBS Cloud supports both automatic and manual journal entries.

### Automatic Journal Entries

The system automatically creates journal entries for:

- Loan disbursements
- Loan repayments
- Savings deposits and withdrawals
- Fee collections
- Interest postings
- Internal transfers

These entries are created automatically when you process the corresponding transaction in the system.

### Manual Journal Entries

Create manual journal entries for transactions not captured by automatic processes:

<Tabs>
  <Tab title="Single Transaction">
    **Single Transaction Entry** - Simple debit/credit entry
    
    <Steps>
      <Step title="Open Transaction Modal">
        Navigate to **Accounting** → **Accounting Entries** and click **Create Transaction** → **Single**.
      </Step>

      <Step title="Enter Transaction Details">
        Complete the transaction form:
        
        **Debit Account**
        - Search and select the account to debit
        - Only accounts with manual transaction permission appear
        
        **Credit Account**
        - Search and select the account to credit
        - Must be different from debit account
        
        **Amount**
        - Enter transaction amount
        - Must be positive number
        
        **Date**
        - Transaction date (defaults to current date)
        - Affects which period the entry appears in
        
        **Description**
        - Required transaction description
        - Explain the reason for the entry
        - Include reference numbers if applicable
        
        <Note>
        The debit and credit amounts must be equal. Single transactions are always balanced entries.
        </Note>
      </Step>

      <Step title="Auto-Print Option">
        ☑️ Check **Auto Print** to automatically generate a receipt after posting.
      </Step>

      <Step title="Post Transaction">
        Click **Post** to record the journal entry. The system:
        - Creates the accounting entry
        - Updates account balances
        - Records audit trail
        - Generates receipt (if auto-print selected)
      </Step>
    </Steps>
  </Tab>

  <Tab title="Multiple Transaction">
    **Multiple Transaction Entry** - Use transaction templates for complex entries
    
    <Steps>
      <Step title="Open Multiple Transaction Modal">
        Click **Create Transaction** → **Multiple** from the accounting entries screen.
      </Step>

      <Step title="Select Transaction Template">
        Choose a pre-configured transaction template. Templates define:
        - Multiple debit or credit accounts
        - Standard transaction patterns
        - Account allocation rules
      </Step>

      <Step title="Enter Amounts">
        The template determines which side has multiple accounts:
        
        **Single Debit, Multiple Credits**
        - One debit account (from template)
        - Enter amounts for each credit account
        - Total credits = debit amount
        
        **Multiple Debits, Single Credit**
        - Multiple debit accounts (from template)
        - Enter amounts for each debit account
        - Total debits = credit amount
      </Step>

      <Step title="Complete Transaction Details">
        - **Date**: Transaction date
        - **Description**: Entry description
        - **Auto Print**: Optional receipt generation
        
        The system validates that total debits equal total credits.
      </Step>

      <Step title="Post Transaction">
        Click **Post** to record the complex journal entry.
      </Step>
    </Steps>
  </Tab>
</Tabs>

### Viewing Accounting Entries

Review posted accounting entries:

<Steps>
  <Step title="Access Accounting Entries">
    Navigate to **Accounting** → **Accounting Entries**.
  </Step>

  <Step title="Filter Entries">
    Use filters to find specific entries:
    
    **Date Range**
    - Start date (defaults to beginning of month)
    - End date (defaults to current date)
    
    **Account Filter**
    - Select specific account to view its entries
    - Leave blank to view all entries
    
    Click **Filter** to apply.
  </Step>

  <Step title="Review Entry Details">
    The entries list displays:
    - Date and time
    - Debit account and amount
    - Credit account and amount
    - Description
    - Created by user
    - Entry reference number
    
    Click on an entry to view full details.
  </Step>
</Steps>

### Printing Journal Entries

Generate entry documents:

1. Find the entry in the accounting entries list
2. Click the **Print** icon
3. System generates the journal entry voucher
4. Download or print the document

The voucher shows:
- Entry date and reference
- Debit and credit accounts
- Amounts
- Description
- Created by user

## Account Balances and Reconciliation

### Viewing Account Balances

Check current account balances:

1. Navigate to **Chart of Accounts**
2. Filter by branch if needed
3. Expand account hierarchy
4. View balance next to each account

Balances show:
- Current balance (real-time)
- Debit or credit indicator
- Currency (if multi-currency)

### Running Balance Reports

Generate balance reports:

1. Navigate to **Reports** → **Accounting Reports**
2. Select **Trial Balance** or **Balance Sheet**
3. Choose date range or specific date
4. Select branch (if applicable)
5. Click **Generate**

The report shows:
- All accounts with balances
- Debit and credit totals
- Account hierarchy
- Running totals

<Tip>
Run trial balance regularly to ensure the accounting system is balanced (total debits = total credits).
</Tip>

## Financial Reports

OpenCBS Cloud generates standard financial reports:

### Balance Sheet

Shows financial position at a specific point in time:

- **Assets**: What the institution owns
- **Liabilities**: What the institution owes
- **Equity**: Net worth

Generate balance sheet:
1. Go to **Reports** → **Accounting Reports** → **Balance Sheet**
2. Select balance sheet date
3. Choose branch or consolidated
4. Click **Generate**

### Income Statement (Profit & Loss)

Shows financial performance over a period:

- **Income**: Revenue from operations
- **Expenses**: Costs of operations
- **Net Income**: Profit or loss

Generate income statement:
1. Go to **Reports** → **Accounting Reports** → **Income Statement**
2. Select date range (month, quarter, year)
3. Choose branch or consolidated
4. Click **Generate**

### Trial Balance

Lists all accounts with their debit and credit balances:

- Verifies accounting equation balance
- Shows all account activity
- Foundation for financial statements

Generate trial balance:
1. Go to **Reports** → **Accounting Reports** → **Trial Balance**
2. Select date or date range
3. Choose branch filter
4. Click **Generate**

### General Ledger

Complete transaction history for accounts:

- All entries for selected accounts
- Running balances
- Transaction details

Generate general ledger:
1. Go to **Reports** → **Accounting Reports** → **General Ledger**
2. Select account(s)
3. Choose date range
4. Click **Generate**

## Multi-Currency Accounting

For institutions operating with multiple currencies:

### Currency-Specific Accounts

- Create accounts for each currency
- Specify currency when creating account
- Balances maintained in account currency

### Exchange Rates

- System uses configured exchange rates
- Rates used for currency conversion
- Reports can show in base or foreign currency

### Foreign Exchange Gains/Losses

- Automatically calculated on revaluation
- Posted to designated GL accounts
- Reflects exchange rate changes

## Period Close and Year-End

### Period Closing

Close accounting periods to prevent backdated entries:

1. Navigate to **Accounting** → **Period Close**
2. Select period to close (month, quarter)
3. Run closing validation
4. Post period close

Once closed:
- No entries can be posted to closed period
- Period balances are locked
- Opening balances for next period created

### Year-End Close

Special process at fiscal year end:

1. Close all periods in the fiscal year
2. Generate year-end reports
3. Post income/expense closing entries
4. Transfer net income to retained earnings
5. Create opening balances for new year

<Warning>
Period closing is usually restricted to accounting administrators. Contact your supervisor before attempting to close periods.
</Warning>

## Maker-Checker for Accounting

Some accounting operations require approval:

### Operations Requiring Approval

- Creating new accounts
- Posting manual journal entries (depending on amount)
- Modifying account structure
- Period closing

### Approval Process

1. Maker creates transaction or account
2. Entry appears in **Maker-Checker** → **Accounting** queue
3. Checker reviews and approves or rejects
4. Approved items become effective
5. Rejected items return to maker for correction

## Troubleshooting

<Warning>
**Common Accounting Issues:**

**"Account not found"**
- Verify account exists in chart of accounts
- Check branch filter
- Ensure account is active

**"Debit and credit must be equal"**
- Transaction is unbalanced
- Verify amounts entered correctly
- Check for rounding errors

**"Account does not allow manual transactions"**
- Account permissions prevent manual entries
- Edit account to allow manual transactions
- Or use appropriate automated transaction

**"Cannot post to closed period"**
- Attempting to post to closed accounting period
- Contact administrator to reopen period if needed
- Post to current period instead
</Warning>
