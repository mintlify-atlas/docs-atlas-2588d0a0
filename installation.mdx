## Overview

OpenCBS Cloud is a flexible, scalable Core Banking System built with Spring Boot and Angular. This guide will walk you through the complete installation process, from system prerequisites to running your first instance.

<Note>
  OpenCBS Cloud consists of two main components: a **Java/Spring Boot backend** and an **Angular frontend**. Both are built together into a single deployable JAR file.
</Note>

## Prerequisites Installation

<Steps>
  <Step title="Install Java Development Kit (JDK)">
    OpenCBS Cloud requires Java 8 or higher.

    <CodeGroup>
    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install openjdk-8-jdk
    java -version
    ```

    ```bash macOS
    brew install openjdk@8
    java -version
    ```

    ```bash Windows
    # Download from Oracle or OpenJDK
    # https://adoptopenjdk.net/
    # Verify installation:
    java -version
    ```
    </CodeGroup>

    <Tip>
      Set the `JAVA_HOME` environment variable to point to your JDK installation directory.
    </Tip>
  </Step>

  <Step title="Install Apache Maven">
    Maven is required to build the OpenCBS Cloud modules.

    <CodeGroup>
    ```bash Ubuntu/Debian
    sudo apt install maven
    mvn -version
    ```

    ```bash macOS
    brew install maven
    mvn -version
    ```

    ```bash Windows
    # Download from Apache Maven
    # https://maven.apache.org/download.cgi
    # Extract and add to PATH
    mvn -version
    ```
    </CodeGroup>
  </Step>

  <Step title="Install PostgreSQL">
    OpenCBS Cloud uses PostgreSQL as its database.

    <CodeGroup>
    ```bash Ubuntu/Debian
    sudo apt install postgresql postgresql-contrib
    sudo systemctl start postgresql
    sudo systemctl enable postgresql
    ```

    ```bash macOS
    brew install postgresql
    brew services start postgresql
    ```

    ```bash Windows
    # Download installer from PostgreSQL
    # https://www.postgresql.org/download/windows/
    ```
    </CodeGroup>
  </Step>

  <Step title="Install Node.js and Yarn">
    Required for building the Angular frontend.

    <CodeGroup>
    ```bash Ubuntu/Debian
    curl -fsSL https://deb.nodesource.com/setup_14.x | sudo -E bash -
    sudo apt install -y nodejs
    npm install -g yarn
    node -v && yarn -v
    ```

    ```bash macOS
    brew install node
    npm install -g yarn
    node -v && yarn -v
    ```

    ```bash Windows
    # Download from nodejs.org
    # https://nodejs.org/
    # Then install Yarn:
    npm install -g yarn
    ```
    </CodeGroup>
  </Step>
</Steps>

## Database Setup

<Steps>
  <Step title="Create database and user">
    Connect to PostgreSQL and create a database for OpenCBS Cloud:

    ```sql
    -- Connect to PostgreSQL
    sudo -u postgres psql

    -- Create database
    CREATE DATABASE opencbs_db;

    -- Create user
    CREATE USER opencbs_user WITH ENCRYPTED PASSWORD 'your_secure_password';

    -- Grant privileges
    GRANT ALL PRIVILEGES ON DATABASE opencbs_db TO opencbs_user;

    -- Exit
    \q
    ```

    <Warning>
      Replace `your_secure_password` with a strong password. Store it securely as you'll need it for the application configuration.
    </Warning>
  </Step>

  <Step title="Configure database connection">
    OpenCBS Cloud uses Spring Boot's auto-configuration. Create an `application.properties` file in your deployment directory:

    ```properties
    # Database Configuration
    spring.datasource.url=jdbc:postgresql://localhost:5432/opencbs_db
    spring.datasource.username=opencbs_user
    spring.datasource.password=your_secure_password
    spring.datasource.driver-class-name=org.postgresql.Driver

    # JPA/Hibernate Configuration
    spring.jpa.database=POSTGRESQL
    spring.jpa.show-sql=false
    spring.jpa.hibernate.ddl-auto=validate

    # Flyway Migration
    spring.flyway.enabled=true
    spring.flyway.baseline-on-migrate=true

    # Server Configuration
    server.port=8080
    ```

    <Tip>
      OpenCBS Cloud uses Flyway for database migrations. The schema will be created automatically on first startup.
    </Tip>
  </Step>

  <Step title="Verify database connection">
    Test the connection to ensure everything is configured correctly:

    ```bash
    psql -h localhost -U opencbs_user -d opencbs_db
    ```

    Enter your password when prompted. If successful, you'll see the PostgreSQL prompt.
  </Step>
</Steps>

## Building from Source

OpenCBS Cloud provides a build script that compiles all modules and packages them into a single JAR file.

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/opencbs/opencbs-cloud.git
    cd opencbs-cloud
    ```
  </Step>

  <Step title="Review the build script">
    The `build.sh` script orchestrates the entire build process:

    ```bash build.sh
    #!/bin/bash

    show_usage() {
        echo -e "Usage: $0 [ARGUMENTS]"
        echo ""
        echo "arguments:"
        echo -e "\t-build version - specify the version of JAR. For example, 1.0.0"
        echo -e "\t-instance name - specify the instance name without opencbs prefix. For example, my-finance or impact-finance"
    }

    if [ "$#" -ne 2 ]; then
        show_usage
        exit 1
    else
        mvn -f ./server/opencbs-spring-boot-starter/pom.xml clean install -DskipTests
        mvn -f ./server/opencbs-core/pom.xml clean install -DskipTests
        mvn -f ./server/opencbs-loans/pom.xml clean install -DskipTests
        mvn -f ./server/opencbs-borrowings/pom.xml clean install -DskipTests
        mvn -f ./server/opencbs-savings/pom.xml clean install -DskipTests
        mvn -f ./server/opencbs-term-deposits/pom.xml clean install -DskipTests
        mvn -f ./server/opencbs-bonds/pom.xml clean install -DskipTests
        mvn -f ./server/opencbs-$2/pom_backend.xml clean install jar:jar -DBUILD_VERSION=$1
    fi
    ```

    <Note>
      The build script requires two arguments:
      1. **Version number** (e.g., 1.0.0)
      2. **Instance name** without the "opencbs" prefix (e.g., server)
    </Note>
  </Step>

  <Step title="Run the build">
    Execute the build script with your desired version and instance name:

    ```bash
    chmod +x build.sh
    ./build.sh 1.0.0 server
    ```

    This process will:
    1. Clean previous builds
    2. Install the Spring Boot starter module
    3. Build the core module
    4. Build all product modules (loans, savings, term deposits, bonds, borrowings)
    5. Build the frontend Angular application
    6. Package everything into a single JAR file

    <Tip>
      The build process can take 10-20 minutes depending on your system. Maven will download all required dependencies on the first build.
    </Tip>
  </Step>

  <Step title="Locate the build artifact">
    After successful build, find the JAR file:

    ```bash
    ls -lh server/opencbs-server/target/*.jar
    ```

    You should see a file named `opencbs-server-1.0.0.jar` (or your specified version).
  </Step>
</Steps>

## Module Architecture

OpenCBS Cloud is built with a modular architecture. Understanding the modules helps with customization and troubleshooting:

<AccordionGroup>
  <Accordion title="opencbs-spring-boot-starter" icon="rocket">
    Base Spring Boot configuration and starter dependencies. This module provides:
    - Spring Boot parent configuration
    - Common dependencies (Spring Data JPA, Security, Web)
    - Base configuration properties
    
    **Version:** Spring Boot 1.5.4
  </Accordion>

  <Accordion title="opencbs-core" icon="cube">
    Core banking functionality including:
    - User management and authentication (JWT-based)
    - Profile management (Person, Company, Group)
    - Branch management
    - Accounting engine
    - Custom fields framework
    - Audit trail
    - Day closure operations
    - Maker-checker workflow
    
    **Key Dependencies:**
    - PostgreSQL driver
    - Flyway for migrations (v4.0.3)
    - Hibernate Envers for auditing (v5.4.1)
    - Jasper Reports (v6.10.0)
    - Swagger for API documentation (v2.9.2)
  </Accordion>

  <Accordion title="opencbs-loans" icon="hand-holding-dollar">
    Loan management module:
    - Loan products configuration
    - Loan applications
    - Loan disbursement
    - Repayment processing
    - Schedule generation and management
    - Collateral management
    - Guarantor management
    - Credit committee workflow
    - Provisioning
  </Accordion>

  <Accordion title="opencbs-savings" icon="piggy-bank">
    Savings account management:
    - Savings products
    - Account opening and management
    - Deposit and withdrawal operations
    - Interest calculation
    - Till management
  </Accordion>

  <Accordion title="opencbs-term-deposits" icon="calendar">
    Term deposit products:
    - Fixed-term deposit products
    - Maturity management
    - Interest accrual
  </Accordion>

  <Accordion title="opencbs-borrowings" icon="building-columns">
    Institutional borrowings:
    - Borrowing products
    - Repayment schedules
    - Interest management
  </Accordion>

  <Accordion title="opencbs-bonds" icon="certificate">
    Bond management:
    - Bond products
    - Issuance tracking
    - Coupon payments
  </Accordion>

  <Accordion title="opencbs-server" icon="server">
    Main application module that:
    - Aggregates all modules
    - Bundles the Angular frontend
    - Provides the executable JAR
    - Includes JasperReports compilation
    
    The build process:
    1. Runs `yarn` to install frontend dependencies
    2. Executes `yarn build-prod` to build Angular app
    3. Copies the `dist/` folder to Spring Boot's static resources
    4. Packages everything into a single JAR
  </Accordion>
</AccordionGroup>

## Running OpenCBS Cloud

<Steps>
  <Step title="Prepare the deployment directory">
    Create a directory for your OpenCBS Cloud deployment:

    ```bash
    mkdir -p /opt/opencbs
    cd /opt/opencbs
    ```

    Copy the JAR file and create the configuration file:

    ```bash
    cp ~/opencbs-cloud/server/opencbs-server/target/opencbs-server-1.0.0.jar .
    ```
  </Step>

  <Step title="Create application configuration">
    Create `application.properties` in the same directory:

    ```properties
    # Database
    spring.datasource.url=jdbc:postgresql://localhost:5432/opencbs_db
    spring.datasource.username=opencbs_user
    spring.datasource.password=your_secure_password

    # JPA
    spring.jpa.hibernate.ddl-auto=validate
    spring.jpa.show-sql=false

    # Flyway
    spring.flyway.enabled=true
    spring.flyway.baseline-on-migrate=true

    # Server
    server.port=8080

    # JWT Security
    jwt.secret=your-jwt-secret-key-change-this-in-production
    jwt.expires_in=86400

    # File Upload
    spring.servlet.multipart.max-file-size=10MB
    spring.servlet.multipart.max-request-size=10MB

    # Logging
    logging.level.com.opencbs=INFO
    logging.level.org.springframework=WARN
    ```

    <Warning>
      **Security Note:** Change the `jwt.secret` to a strong, random value in production. This is used to sign authentication tokens.
    </Warning>
  </Step>

  <Step title="Start the application">
    Run OpenCBS Cloud:

    ```bash
    java -jar opencbs-server-1.0.0.jar
    ```

    For production deployment with custom memory settings:

    ```bash
    java -Xms512m -Xmx2048m -jar opencbs-server-1.0.0.jar
    ```

    <Tip>
      Add `--spring.config.location=./application.properties` if your config file is in a different location.
    </Tip>
  </Step>

  <Step title="Verify the installation">
    Check that OpenCBS Cloud is running:

    ```bash
    # Check if the server is responding
    curl http://localhost:8080/api/info

    # Check logs
    tail -f logs/spring.log
    ```

    You should see the Spring Boot banner and initialization logs indicating successful startup.
  </Step>

  <Step title="Access the web interface">
    Open your browser and navigate to:

    ```
    http://localhost:8080
    ```

    You should see the OpenCBS Cloud login page.

    **Default Credentials:**
    - Username: `admin`
    - Password: (set during initial database setup)

    <Warning>
      Change the default admin password immediately after first login!
    </Warning>
  </Step>
</Steps>

## Production Deployment

For production environments, consider these additional steps:

<Steps>
  <Step title="Create a system service">
    Create a systemd service file for automatic startup:

    ```bash
    sudo nano /etc/systemd/system/opencbs.service
    ```

    Add the following content:

    ```ini
    [Unit]
    Description=OpenCBS Cloud Banking System
    After=network.target postgresql.service

    [Service]
    Type=simple
    User=opencbs
    WorkingDirectory=/opt/opencbs
    ExecStart=/usr/bin/java -Xms512m -Xmx2048m -jar opencbs-server-1.0.0.jar
    Restart=on-failure
    RestartSec=10

    [Install]
    WantedBy=multi-user.target
    ```

    Enable and start the service:

    ```bash
    sudo systemctl daemon-reload
    sudo systemctl enable opencbs
    sudo systemctl start opencbs
    sudo systemctl status opencbs
    ```
  </Step>

  <Step title="Configure reverse proxy (Nginx)">
    Install and configure Nginx as a reverse proxy:

    ```nginx
    server {
        listen 80;
        server_name opencbs.yourdomain.com;

        location / {
            proxy_pass http://localhost:8080;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        client_max_body_size 10M;
    }
    ```

    For HTTPS (recommended):

    ```bash
    sudo apt install certbot python3-certbot-nginx
    sudo certbot --nginx -d opencbs.yourdomain.com
    ```
  </Step>

  <Step title="Configure database backups">
    Set up automated PostgreSQL backups:

    ```bash
    #!/bin/bash
    # /opt/opencbs/backup.sh
    
    BACKUP_DIR="/opt/opencbs/backups"
    TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
    
    mkdir -p $BACKUP_DIR
    
    pg_dump -h localhost -U opencbs_user opencbs_db | \
      gzip > $BACKUP_DIR/opencbs_backup_$TIMESTAMP.sql.gz
    
    # Keep only last 30 days of backups
    find $BACKUP_DIR -name "opencbs_backup_*.sql.gz" -mtime +30 -delete
    ```

    Add to crontab for daily backups:

    ```bash
    crontab -e
    # Add this line:
    0 2 * * * /opt/opencbs/backup.sh
    ```
  </Step>

  <Step title="Enable monitoring">
    OpenCBS Cloud includes Spring Boot Actuator endpoints:

    ```properties
    # application.properties
    management.endpoints.web.exposure.include=health,info,metrics
    management.endpoint.health.show-details=when-authorized
    ```

    Access monitoring endpoints:
    - Health: `http://localhost:8080/actuator/health`
    - Metrics: `http://localhost:8080/actuator/metrics`
  </Step>
</Steps>

    spring.datasource.url=jdbc:postgresql://staging-db:5432/opencbs_staging

    spring.jpa.show-sql=false
    logging.level.com.opencbs=WARN

    jwt.expires_in=28800

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build fails with 'Could not resolve dependencies'">
    **Solution:**
    - Check your internet connection (Maven needs to download dependencies)
    - Clear Maven cache: `rm -rf ~/.m2/repository`
    - Try running with `-U` flag: `mvn clean install -U`
    - Check if Maven central repository is accessible
  </Accordion>

  <Accordion title="Database connection fails on startup">
    **Solution:**
    - Verify PostgreSQL is running: `sudo systemctl status postgresql`
    - Check connection parameters in `application.properties`
    - Test connection: `psql -h localhost -U opencbs_user -d opencbs_db`
    - Check PostgreSQL logs: `sudo tail -f /var/log/postgresql/postgresql-*.log`
    - Ensure `pg_hba.conf` allows connections from your application
  </Accordion>

  <Accordion title="Frontend build fails with 'yarn build-prod' error">
    **Solution:**
    - Ensure Node.js and Yarn are installed: `node -v && yarn -v`
    - Clear node_modules: `cd client && rm -rf node_modules && yarn install`
    - Check if you have enough memory (requires ~4GB for Angular build)
    - Try building manually: `cd client && yarn build-prod`
  </Accordion>

  <Accordion title="Port 8080 already in use">
    **Solution:**
    - Check what's using port 8080: `sudo lsof -i :8080`
    - Change the port in `application.properties`: `server.port=8081`
    - Kill the process using the port: `sudo kill -9 <PID>`
  </Accordion>

  <Accordion title="Application starts but shows 404 for all pages">
    **Solution:**
    - This usually means the Angular frontend wasn't bundled correctly
    - Check if `static` folder exists in the JAR: `jar tf opencbs-server-1.0.0.jar | grep static`
    - Rebuild with `mvn clean install` (don't skip tests with `-DskipTests` in final build)
    - Ensure the exec-maven-plugin runs the Yarn build step
  </Accordion>

  <Accordion title="Out of memory errors during runtime">
    **Solution:**
    - Increase JVM memory: `java -Xms1024m -Xmx4096m -jar opencbs-server-1.0.0.jar`
    - Configure heap dump on error: `-XX:+HeapDumpOnOutOfMemoryError`
    - Monitor with: `jmap -heap <pid>`
    - Consider using G1GC: `-XX:+UseG1GC`
  </Accordion>
</AccordionGroup>
