The Holidays API provides endpoints for creating, updating, and retrieving holiday information. Holidays are used by the system to calculate loan schedules, payment due dates, and other date-sensitive operations.

## Get All Holidays

<CodeGroup>
```http Request
GET /api/holidays
```
</CodeGroup>

Retrieves a list of all configured holidays in the system.

### Response

Returns an array of holiday objects.

<ResponseField name="array" type="array">
  Array of holiday objects
  <Expandable title="Holiday Object">
    <ResponseField name="id" type="long">
      Unique identifier for the holiday
    </ResponseField>
    <ResponseField name="name" type="string">
      Name of the holiday (e.g., "New Year's Day", "Christmas")
    </ResponseField>
    <ResponseField name="date" type="string">
      Date of the holiday in ISO 8601 format (YYYY-MM-DD)
    </ResponseField>
    <ResponseField name="isAnnual" type="boolean">
      Whether this holiday recurs annually on the same date. If true, the system will automatically apply this holiday every year.
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>
```json Response Example

  },
  {
    "id": 3,

    "isAnnual": true
  },

    "id": 4,
```

</CodeGroup>

## Usage Examples

### Create an annual holiday

<CodeGroup>
```bash cURL
curl -X POST "https://api.opencbs.com/api/holidays" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "National Day",
    "date": "2026-08-15",
    "isAnnual": true
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/holidays', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'National Day',
    date: '2026-08-15',
    isAnnual: true
  })
});

const holiday = await response.json();
console.log('Created holiday:', holiday);
```

```python Python
import requests

response = requests.post(
    'https://api.opencbs.com/api/holidays',
    headers={
        'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
        'Content-Type': 'application/json'
    },
    json={
        'name': 'National Day',
        'date': '2026-08-15',
        'isAnnual': True
    }
)

holiday = response.json()
print('Created holiday:', holiday)
```
</CodeGroup>

### Get all annual holidays

<CodeGroup>
```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/holidays', {
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});

const holidays = await response.json();
const annualHolidays = holidays.filter(h => h.isAnnual);

console.log('Annual holidays:', annualHolidays);
```

```python Python
import requests

response = requests.get(
    'https://api.opencbs.com/api/holidays',
    headers={'Authorization': 'Bearer YOUR_ACCESS_TOKEN'}
)

holidays = response.json()
annual_holidays = [h for h in holidays if h['isAnnual']]

print('Annual holidays:', annual_holidays)
```
</CodeGroup>

## How Holidays Affect the System

### Loan Schedules

When calculating loan repayment schedules, the system automatically skips holidays when determining due dates:

- If a payment is due on a holiday, it's automatically moved to the next business day
- Annual holidays are considered for all future years in long-term loans
- This ensures borrowers are never expected to make payments on non-working days

### Interest Calculations

Depending on your system configuration, holidays may affect:

- Daily interest accrual calculations
- Late fee calculations
- Grace period computations

### Business Day Calculations

Holidays are excluded when the system calculates:

- Processing timeframes (e.g., "5 business days")
- Maturity dates
- Disbursement schedules

## Best Practices

### Annual vs. One-time Holidays

<ParamField body="isAnnual" type="boolean">
  - Set to `true` for holidays that occur on the same date every year (e.g., New Year's Day, Christmas)
  - Set to `false` for one-time events (e.g., special bank closures, elections)
</ParamField>

### Planning Ahead

- Configure holidays at the beginning of each year
- For annual holidays, you only need to create them once
- Review and update one-time holidays annually

### Date Format

Always use ISO 8601 date format (YYYY-MM-DD):
- Correct: `"2026-12-25"`
- Incorrect: `"12/25/2026"` or `"25-12-2026"`

---

## Validation Rules

### Create and Update

- `name` is required and cannot be empty
- `date` is required and must be a valid ISO 8601 date
- `isAnnual` is required and must be a boolean value
- Duplicate holidays (same date) may be allowed depending on system configuration
