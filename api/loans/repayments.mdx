## Overview

The Loan Repayments API handles all aspects of processing borrower payments, including:

- Calculating optimal payment splits across principal, interest, and penalties
- Previewing how repayments affect loan schedules
- Processing actual repayments through maker-checker workflow
- Supporting multiple repayment types and methods
- Handling early repayments and overpayments

All repayment operations require the loan to be actualized to the repayment date to ensure accurate calculations.

## Repayment Types

OpenCBS supports several repayment types:

- **NORMAL_REPAYMENT**: Standard scheduled repayment
- **EARLY_TOTAL_REPAYMENT**: Full loan payoff before maturity
- **EARLY_PARTIAL_REPAYMENT**: Extra payment reducing principal
- **PENALTY_REPAYMENT**: Payment specifically for penalties
- **RESCHEDULED_REPAYMENT**: Repayment after loan rescheduling

Each type has different calculation rules for splitting payments across principal, interest, and fees.

### Calculate Repayment Split

<CodeGroup>
```bash cURL
curl -X POST "https://your-instance.opencbs.com/api/loans/123/repayment/split" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "timestamp": "2024-03-15T14:30:00",
    "total": 1000.00,
    "repaymentType": "NORMAL_REPAYMENT",
    "paymentMethod": 1
  }'
```

```
POST /api/loans/{loanId}/repayment/split
```

Calculate how a repayment amount should be split across principal, interest, penalties, and fees based on the loan's current state and repayment type.

#### Path Parameters

<ParamField path="loanId" type="integer" required>
  Loan ID
</ParamField>

## Complete Repayment Workflow

Processing a repayment involves three steps:

<Steps>
  <Step title="Calculate Split">
    Call `/api/loans/{loanId}/repayment/split` with the repayment amount to get the optimal split across principal, interest, and penalties.
    
    ```bash
    POST /api/loans/123/repayment/split
    {
      "timestamp": "2024-03-15T14:30:00",
      "total": 1000.00,
      "repaymentType": "NORMAL_REPAYMENT",
      "paymentMethod": 1
    }
    ```
  </Step>
  
  <Step title="Preview Schedule">
    Call `/api/loans/{loanId}/repayment/preview` with the split values to show the borrower how their loan will be updated.
    
    ```bash
    POST /api/loans/123/repayment/preview
    {
      "timestamp": "2024-03-15T14:30:00",
      "total": 1000.00,
      "principal": 788.49,
      "interest": 100.00,
      "penalty": 111.51,
      "earlyRepaymentFee": 0.00,
      "repaymentType": "NORMAL_REPAYMENT",
      "paymentMethod": 1
    }
    ```
  </Step>
  
  <Step title="Process Repayment">
    Call `/api/loans/{loanId}/repayment/repay` with the same split values to create the maker-checker request.
    
    ```bash
    POST /api/loans/123/repayment/repay
    {
      "timestamp": "2024-03-15T14:30:00",
      "total": 1000.00,
      "principal": 788.49,
      "interest": 100.00,
      "penalty": 111.51,
      "earlyRepaymentFee": 0.00,
      "repaymentType": "NORMAL_REPAYMENT",
      "paymentMethod": 1
    }
    ```
  </Step>
  
  <Step title="Approve Request">
    An authorized checker must approve the maker-checker request before the repayment is posted to the loan. See the [User Roles & Permissions](/admin/user-roles-permissions) guide for more on the maker-checker workflow.
  </Step>
</Steps>

## Repayment Types Explained

### Normal Repayment

Used for regular scheduled payments. Payment is split according to the repayment priority:

1. Overdue penalties
2. Overdue interest
3. Overdue principal
4. Current interest
5. Current principal

```json
{
  "repaymentType": "NORMAL_REPAYMENT"
}
```

### Early Total Repayment

Complete loan payoff before maturity. Includes:

- All outstanding principal
- All accrued interest
- All penalties
- Early repayment fee (if configured in product)

```json
{
  "repaymentType": "EARLY_TOTAL_REPAYMENT"
}
```

<Note>
  The system automatically calculates the early repayment fee based on the product configuration.
</Note>

### Early Partial Repayment

Extra principal payment that reduces the loan balance. The system will:

1. Apply payment to penalties and interest first
2. Apply remaining amount to principal
3. Recalculate future installments based on new OLB
4. Charge early partial repayment fee (if configured)

```json
{
  "repaymentType": "EARLY_PARTIAL_REPAYMENT"
}
```

### Penalty Repayment

Payment specifically for outstanding penalties without affecting principal or interest schedule.

```json
{
  "repaymentType": "PENALTY_REPAYMENT"
}
```

## Best Practices

### Repayment Processing

1. **Always Calculate Split First**: Never manually calculate splits - always use the `/split` endpoint
2. **Preview Before Processing**: Show borrowers the preview schedule before finalizing repayment
3. **Actualize Loans**: Ensure loans are actualized to the repayment date before processing
4. **Validate Amounts**: Check that repayment amounts are reasonable and within expected ranges
5. **Receipt Generation**: Use `autoPrint: true` for cash transactions to generate receipts immediately

### Error Handling

1. **Actualization Errors**: If you receive an actualization error, call the actualize endpoint first
2. **Invalid Amounts**: Handle validation errors for amounts that exceed maximum or are below minimum
3. **Payment Method**: Ensure payment method IDs are valid for your installation
4. **Date Validation**: Verify repayment dates are not in the future or before loan disbursement

### Performance

1. **Batch Processing**: For multiple repayments, process them sequentially to avoid race conditions
2. **Caching**: Cache payment method lookups and product configurations
3. **Timeout Handling**: Set appropriate timeouts for actualization operations which can be slow

### Security

1. **Amount Verification**: Implement double-entry verification for large repayment amounts
2. **Audit Trail**: Log all repayment calculations and preview operations
3. **Maker-Checker**: Never bypass the maker-checker workflow for repayments
4. **Receipt Safeguards**: Ensure receipts are only generated after approval

## Common Scenarios

### Scenario 1: Regular Monthly Payment

```javascript
// Step 1: Calculate split
const split = await calculateSplit(loanId, {
  timestamp: '2024-03-15T10:00:00',
  total: 888.49,
  repaymentType: 'NORMAL_REPAYMENT',
  paymentMethod: 1 // Cash
});

// Step 2: Preview (optional)
const preview = await previewRepayment(loanId, split);

// Step 3: Process
const request = await processRepayment(loanId, split);
console.log(`Created request ${request.id} for approval`);
```

### Scenario 2: Early Loan Payoff

```javascript
// Step 1: Calculate total payoff amount
const split = await calculateSplit(loanId, {
  timestamp: '2024-03-15T10:00:00',
  total: 9500.00, // Estimated total
  repaymentType: 'EARLY_TOTAL_REPAYMENT',
  paymentMethod: 2 // Bank transfer
});

// The split response includes the exact amount needed
console.log(`Total payoff: ${split.total}`);
console.log(`Early repayment fee: ${split.earlyRepaymentFee}`);

// Step 2: Show borrower the breakdown
const preview = await previewRepayment(loanId, split);

// Step 3: Process when borrower confirms
const request = await processRepayment(loanId, split);
```

### Scenario 3: Overpayment

```javascript
// Borrower pays more than scheduled installment
const split = await calculateSplit(loanId, {
  timestamp: '2024-03-15T10:00:00',
  total: 1500.00, // More than $888.49 installment
  repaymentType: 'EARLY_PARTIAL_REPAYMENT',
  paymentMethod: 1
});

// System automatically applies extra to principal
console.log(`Principal: ${split.principal}`);
console.log(`Interest: ${split.interest}`);

// Preview shows reduced future payments or shorter term
const preview = await previewRepayment(loanId, split);
```

### Scenario 4: Partial Payment

```javascript
// Borrower pays less than full installment
const split = await calculateSplit(loanId, {
  timestamp: '2024-03-15T10:00:00',
  total: 500.00, // Less than $888.49 installment
  repaymentType: 'NORMAL_REPAYMENT',
  paymentMethod: 1
});

// System prioritizes interest and penalties
// Remaining installment becomes overdue
const preview = await previewRepayment(loanId, split);
// Preview shows partial payment applied
```
