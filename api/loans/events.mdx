## Overview

Loan Events provide a complete audit trail of all activities and transactions on a loan, including:

- Disbursements and repayments
- Interest and penalty accruals
- Rescheduling and restructuring
- Write-offs and reversals
- Other fees and charges
- Status changes

Events are immutable records that enable comprehensive reporting, reconciliation, and compliance tracking.

## Event Types

OpenCBS tracks numerous event types throughout the loan lifecycle:

### Financial Events

- **DISBURSEMENT**: Loan funds disbursed to borrower
- **REPAYMENT**: Payment received from borrower
- **ACCRUAL_OF_INTEREST**: Daily interest accrual
- **ACCRUAL_OF_PENALTY**: Penalty accrual for overdue amounts
- **WRITE_OFF_PRINCIPAL**: Principal written off
- **WRITE_OFF_INTEREST**: Interest written off
- **WRITE_OFF_PENALTY**: Penalty written off
- **OTHER_FEE**: Additional fee charged

### Operational Events

- **RESCHEDULE**: Loan schedule modified
- **TOP_UP**: Additional funds added to loan
- **ROLLBACK**: Transaction reversed
- **LOAN_CLOSE**: Loan closed

### Provision Events

- **SPECIFIC_PROVISION**: Specific provision applied
- **PROVISION_REVERSAL**: Provision reversed

## Event Grouping

Events are grouped together when they result from a single business transaction. The `groupKey` field links related events.

### Disbursement Group

A loan disbursement typically creates multiple events:

```json

  {
    "id": 1001,
    "eventType": "DISBURSEMENT",
    "amount": 10000.00,
    "groupKey": 100
  },
```

## Use Cases

### Transaction History

Display a borrower's transaction history:

```javascript
// Get consolidated events for cleaner display
const events = await fetch(
  `/api/loans/${loanId}/events/group?fromDate=${startDate}&toDate=${endDate}`
).then(r => r.json());

// Display one row per transaction
events.forEach(event => {
  console.log(`${event.effectiveAt}: ${event.eventType} - ${event.amount}`);
});
```

### Loan Statement Generation

Generate a detailed loan statement:

```javascript
// Get all events for the statement period
const events = await fetch(
  `/api/loans/${loanId}/events?fromDate=${statementStart}&toDate=${statementEnd}`
).then(r => r.json());

// Group by date and event type
const statement = events.reduce((acc, event) => {
  const date = event.effectiveAt.split('T')[0];
  if (!acc[date]) acc[date] = [];
  acc[date].push(event);
  return acc;
}, {});

// Generate statement rows
Object.entries(statement).forEach(([date, dayEvents]) => {
  console.log(`\nDate: ${date}`);
  dayEvents.forEach(e => {
    console.log(`  ${e.eventType}: ${e.amount}`);
  });
});
```

### Reconciliation

Reconcile payments against accounting system:

```javascript
// Get all repayment events for a period
const repayments = await fetch(
  `/api/loans/${loanId}/events?eventType=REPAYMENT&fromDate=${monthStart}&toDate=${monthEnd}`
).then(r => r.json());

// Sum by group key to get transaction totals
const transactions = repayments.reduce((acc, event) => {
  if (!acc[event.groupKey]) {
    acc[event.groupKey] = {
      date: event.effectiveAt,
      total: 0,
      events: []
    };
  }
  acc[event.groupKey].total += Math.abs(event.amount);
  acc[event.groupKey].events.push(event);
  return acc;
}, {});

// Compare with external payment records
Object.values(transactions).forEach(txn => {
  console.log(`Transaction on ${txn.date}: $${txn.total}`);
});
```

### Audit Trail

Track who made changes and when:

```javascript
// Get all events with full details
const events = await fetch(
  `/api/loans/${loanId}/events`
).then(r => r.json());

// Filter for manual events (not system-generated)
const manualEvents = events.filter(
  e => e.createdBy.username !== 'system'
);

// Show audit trail
manualEvents.forEach(event => {
  console.log(
    `${event.createdAt}: ${event.createdBy.firstName} ${event.createdBy.lastName} ` +
    `performed ${event.eventType} for ${event.amount}`
  );
});
```

### Interest Accrual Report

Track interest accrued over time:

```javascript
// Get all interest accrual events
const accruals = await fetch(
  `/api/loans/${loanId}/events?eventType=ACCRUAL_OF_INTEREST&fromDate=${yearStart}`
).then(r => r.json());

// Calculate total accrued
const totalAccrued = accruals.reduce(
  (sum, event) => sum + event.amount,
  0
);

console.log(`Total interest accrued: $${totalAccrued.toFixed(2)}`);

// Show daily accrual
accruals.forEach(event => {
  console.log(`${event.effectiveAt.split('T')[0]}: $${event.amount}`);
});
```

## Best Practices

### Query Optimization

1. **Use Date Filters**: Always filter by date range to limit result set size
2. **Event Type Filtering**: Filter by event type when you only need specific events
3. **Grouped vs. Detailed**: Use `/events/group` for summaries, `/events` for details
4. **Pagination**: For large date ranges, implement pagination on the client side

### Data Interpretation

1. **Amount Signs**: Remember positive = inflow/accrual, negative = outflow/payment
2. **Deleted Events**: Check the `deleted` flag to filter out reversed transactions
3. **System Events**: Filter by `createdBy` to distinguish manual vs. automated events
4. **Effective vs. Created**: Use `effectiveAt` for financial reporting, `createdAt` for audit trails

### Performance

1. **Cache Results**: Event data is immutable once created, safe to cache
2. **Incremental Updates**: Query only new events since last fetch using `fromDate`
3. **Local Aggregation**: Download raw events and aggregate locally for complex reports
4. **Avoid Large Ranges**: Limit queries to necessary date ranges

### Compliance

1. **Complete History**: Events provide the complete audit trail for regulatory compliance
2. **Immutability**: Events cannot be modified, only marked as deleted through reversals
3. **User Tracking**: All events track who created them for accountability
4. **Timestamp Accuracy**: Both creation and effective timestamps are recorded
