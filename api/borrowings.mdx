## Overview

The Borrowings API enables financial institutions to manage borrowing accounts (institutional loans), where the organization borrows money from external sources. This is distinct from loans where the institution lends to clients.

## Borrowings

### Create Borrowing

Create a new borrowing account in PENDING status.

<CodeGroup>
```bash cURL
curl -X POST https://api.opencbs.com/api/borrowings \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "borrowingProductId": 10,
    "profileId": 456,
    "correspondenceAccountId": 789,
    "amount": 100000,
    "interestRate": 8.5,
    "scheduleType": "ANNUITY",
    "maturity": 24,
    "gracePeriod": 2,
    "preferredRepaymentDate": "2024-02-01",
    "disbursementDate": "2024-01-15"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/borrowings', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    borrowingProductId: 10,
    profileId: 456,
    correspondenceAccountId: 789,
    amount: 100000,
    interestRate: 8.5,
    scheduleType: 'ANNUITY',
    maturity: 24,
    gracePeriod: 2,
    preferredRepaymentDate: '2024-02-01',
    disbursementDate: '2024-01-15'
  })
});
```

```python Python
import requests

response = requests.post(
    'https://api.opencbs.com/api/borrowings',
    headers={
        'Authorization': 'Bearer YOUR_TOKEN',
        'Content-Type': 'application/json'
    },
    json={
        'borrowingProductId': 10,
        'profileId': 456,
        'correspondenceAccountId': 789,
        'amount': 100000,
        'interestRate': 8.5,
        'scheduleType': 'ANNUITY',
        'maturity': 24,
        'gracePeriod': 2,
        'preferredRepaymentDate': '2024-02-01',
        'disbursementDate': '2024-01-15'
    }
)
```
</CodeGroup>

<ParamField body="borrowingProductId" type="number" required>
  ID of the borrowing product to use
</ParamField>

<ParamField body="profileId" type="number" required>
  ID of the lender profile
</ParamField>

<ParamField body="correspondenceAccountId" type="number" required>
  ID of the correspondence account for transactions
</ParamField>

<ParamField body="amount" type="number" required>
  Borrowing amount
</ParamField>

<ParamField body="interestRate" type="number" required>
  Annual interest rate (e.g., 8.5 for 8.5%)
</ParamField>

<ParamField body="scheduleType" type="string" required>
  Repayment schedule type (e.g., ANNUITY, EQUAL_INSTALLMENTS, DECLINING_BALANCE)
</ParamField>

<ParamField body="maturity" type="number" required>
  Loan term in months or installments
</ParamField>

<ParamField body="gracePeriod" type="number">
  Grace period in installments
</ParamField>

<ParamField body="preferredRepaymentDate" type="string">
  Preferred date for repayments (ISO 8601: YYYY-MM-DD)
</ParamField>

<ParamField body="disbursementDate" type="string" required>
  Date when funds will be disbursed (ISO 8601: YYYY-MM-DD)
</ParamField>

### Preview Borrowing Schedule

```bash cURL
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \

    "amount": 50000,
    "interestRate": 7.5,
    "scheduleType": "ANNUITY",
    "maturity": 12,
    "disbursementDate": "2024-01-15"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/borrowings/preview', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    amount: 50000,
    interestRate: 7.5,
    scheduleType: 'ANNUITY',
    maturity: 12,
    disbursementDate: '2024-01-15'
```

### Disburse Borrowing

Activate a PENDING borrowing by disbursing the funds.

```bash
POST /api/borrowings/{borrowingId}/disburse
```

<ParamField path="borrowingId" type="number" required>
  Borrowing ID
</ParamField>

The borrowing must be in PENDING status. After disbursement, status changes to ACTIVE.

---

### Get Borrowing Schedule

Retrieve the current repayment schedule for an active borrowing.

```bash
GET /api/borrowings/{borrowingId}/schedule
```

<ParamField path="borrowingId" type="number" required>
  Borrowing ID
</ParamField>

<Expandable title="Response">
Returns a schedule object with:
- List of installments
- Payment amounts (principal, interest)
- Due dates
- Paid amounts
- Outstanding balances
</Expandable>

### Roll Back Borrowing Event

Roll back the last event on a borrowing account (e.g., undo a repayment).

<CodeGroup>
```bash cURL
curl -X POST https://api.opencbs.com/api/borrowings/123/roll-back \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "comment": "Reversing incorrect payment posted on wrong date"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/borrowings/123/roll-back', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    comment: 'Reversing incorrect payment posted on wrong date'
  })
});
```
</CodeGroup>

<ParamField path="borrowingId" type="number" required>
  Borrowing ID
</ParamField>

<ParamField body="comment" type="string" required>
  Reason for rolling back the event
</ParamField>

---

### Actualize Borrowing

Trigger the actualization process to calculate and apply accrued interest.

```bash
POST /api/borrowings/actualize/{borrowingId}?date={date}
```

<ParamField path="borrowingId" type="number" required>
  Borrowing ID
</ParamField>

<ParamField query="date" type="string" required>
  Date to actualize to (ISO 8601: YYYY-MM-DD)
</ParamField>

Returns: `"Actualize borrowing started"`

### Split Repayment

Calculate how a repayment amount will be split across principal, interest, fees, and penalties.

```bash
POST /api/borrowings/{borrowingId}/repayment/split
```

<ParamField path="borrowingId" type="number" required>
  Borrowing ID
</ParamField>

Request body uses the same structure as preview repayment.

<Expandable title="Response">
<ResponseField name="principal" type="number">

<ResponseField name="interest" type="number">

## Permissions

The following permissions control access to borrowing operations:

- `GET_BORROWINGS` - View borrowing accounts
- `ACTUALIZE_BORROWINGS` - Trigger actualization
- `PAST_REPAYMENTS` - Make repayments with past dates
