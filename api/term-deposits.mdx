## Overview

The Term Deposits API provides endpoints for managing term deposit accounts and products. Term deposits allow clients to deposit funds for a fixed period at a specified interest rate.

## Term Deposits

### Create Term Deposit

<CodeGroup>
```bash cURL
curl -X POST https://api.opencbs.com/api/term-deposits \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "profileId": 123,
    "termDepositProductId": 5,
    "interestRate": 5.5,
    "termAgreement": 12,
    "earlyCloseFeeRate": 1.5,
    "earlyCloseFeeFlat": 10.00,
    "serviceOfficerId": 45
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/term-deposits', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    profileId: 123,
    termDepositProductId: 5,
    interestRate: 5.5,
    termAgreement: 12,
    earlyCloseFeeRate: 1.5,
    earlyCloseFeeFlat: 10.00,
    serviceOfficerId: 45
  })
});
```

```python Python
import requests

response = requests.post(
    'https://api.opencbs.com/api/term-deposits',
    headers={
        'Authorization': 'Bearer YOUR_TOKEN',
        'Content-Type': 'application/json'
    },
    json={
        'profileId': 123,
        'termDepositProductId': 5,
        'interestRate': 5.5,
        'termAgreement': 12,
        'earlyCloseFeeRate': 1.5,
        'earlyCloseFeeFlat': 10.00,
        'serviceOfficerId': 45
    }
)
```
</CodeGroup>

<ParamField body="profileId" type="number" required>
  ID of the profile (client) opening the term deposit
</ParamField>

<ParamField body="termDepositProductId" type="number" required>
  ID of the term deposit product to use
</ParamField>

<ParamField body="interestRate" type="number" required>
  Annual interest rate (e.g., 5.5 for 5.5%)
</ParamField>

<ParamField body="termAgreement" type="number" required>
  Term length in months
</ParamField>

<ParamField body="earlyCloseFeeRate" type="number">
  Percentage fee for early closure
</ParamField>

<ParamField body="earlyCloseFeeFlat" type="number">
  Flat fee amount for early closure
</ParamField>

<ParamField body="serviceOfficerId" type="number">
  ID of the service officer managing this term deposit
</ParamField>

<ParamField body="locked" type="boolean">
  Whether the term deposit is locked (defaults to false)
</ParamField>

### Open Term Deposit

Activate a term deposit with an initial deposit amount.

<CodeGroup>
```bash cURL
curl -X POST https://api.opencbs.com/api/term-deposits/123/open \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "initialAmount=10000&openDate=2024-01-15"
```

```javascript JavaScript
const params = new URLSearchParams({
  initialAmount: '10000',
  openDate: '2024-01-15'
});

const response = await fetch(`https://api.opencbs.com/api/term-deposits/123/open?${params}`, {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
```
</CodeGroup>

<ParamField path="id" type="number" required>
  Term deposit ID
</ParamField>

<ParamField query="initialAmount" type="number" required>
  Initial deposit amount
</ParamField>

<ParamField query="openDate" type="string" required>
  Opening date (ISO 8601 format: YYYY-MM-DD)
</ParamField>

### Actualize Term Deposit

Trigger the actualization process for a term deposit to calculate and apply accrued interest up to a specific date.

```bash
POST /api/term-deposits/actualize/{termDepositId}?date={date}
```

<ParamField path="termDepositId" type="number" required>
  Term deposit ID
</ParamField>

<ParamField query="date" type="string" required>
  Date to actualize to (ISO 8601 format: YYYY-MM-DD)
</ParamField>

Returns: `"Actualize term deposit started"`

## Permissions

The following permissions are required for term deposit operations:

- `TERM_DEPOSIT_CREATE` - Create new term deposits
- `TERM_DEPOSIT_UPDATE` - Update pending term deposits
- `TERM_DEPOSIT_OPEN` - Open term deposits
- `TERM_DEPOSIT_CLOSE` - Close term deposits
- `TERM_DEPOSIT_LOCK` - Lock/unlock term deposits
- `ACTUALIZE_TERM_DEPOSIT` - Trigger actualization
- `MAKER_FOR_TERM_DEPOSIT_PRODUCT` - Create/update products (maker-checker)
