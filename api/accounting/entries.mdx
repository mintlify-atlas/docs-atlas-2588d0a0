title: 'Accounting Entries'
description: 'Create and retrieve accounting entries (journal entries) in the general ledger'
---

Accounting entries represent double-entry bookkeeping transactions in the general ledger. Each entry contains a debit account, credit account, amount, and metadata.

## Double-Entry Accounting

Every accounting entry in OpenCBS follows double-entry bookkeeping principles:

- Each transaction affects at least two accounts
- Debits must equal credits
- One account is debited, another is credited
- The accounting equation (Assets = Liabilities + Equity) is always maintained

## Get Accounting Entries

<CodeGroup>
```bash cURL
curl -X GET "https://your-instance.opencbs.com/api/accounting/entries?page=0&size=20" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://your-instance.opencbs.com/api/accounting/entries?page=0&size=20',
  {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);
const entries = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://your-instance.opencbs.com/api/accounting/entries',
    params={'page': 0, 'size': 20},
    headers={'Authorization': 'Bearer YOUR_TOKEN'}
)
entries = response.json()
```
</CodeGroup>

## Create Accounting Entry

<CodeGroup>
```bash cURL
curl -X POST "https://your-instance.opencbs.com/api/accounting/entry" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "debitAccountId": 25,
    "creditAccountId": 30,
    "amount": 2500.00,
    "description": "Equipment purchase",
    "documentNumber": "EQ-2024-015",
    "createdAt": "2024-03-15T09:00:00",
    "effectiveAt": "2024-03-15T09:00:00"
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://your-instance.opencbs.com/api/accounting/entry',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      debitAccountId: 25,
      creditAccountId: 30,
      amount: 2500.00,
      description: 'Equipment purchase',
      documentNumber: 'EQ-2024-015',
      createdAt: '2024-03-15T09:00:00',
      effectiveAt: '2024-03-15T09:00:00'
    })
  }
);
const entry = await response.json();
```

```python Python
import requests
from datetime import datetime

response = requests.post(
    'https://your-instance.opencbs.com/api/accounting/entry',
    headers={
        'Authorization': 'Bearer YOUR_TOKEN',
        'Content-Type': 'application/json'
    },
    json={
        'debitAccountId': 25,
        'creditAccountId': 30,
        'amount': 2500.00,
        'description': 'Equipment purchase',
        'documentNumber': 'EQ-2024-015',
        'createdAt': '2024-03-15T09:00:00',
        'effectiveAt': '2024-03-15T09:00:00'
    }
)
entry = response.json()
```
</CodeGroup>

### Request Body

<ParamField body="debitAccountId" type="number" required>
  ID of the account to debit
</ParamField>

<ParamField body="creditAccountId" type="number" required>
  ID of the account to credit
</ParamField>

<ParamField body="amount" type="number" required>
  Transaction amount (must be positive)
</ParamField>

<ParamField body="description" type="string" required>
  Description of the transaction
</ParamField>

<ParamField body="documentNumber" type="string">
  Reference document or voucher number
</ParamField>

<ParamField body="createdAt" type="string">
  Entry creation timestamp (ISO 8601 format: YYYY-MM-DDTHH:mm:ss). Defaults to current time if not provided.
</ParamField>

<ParamField body="effectiveAt" type="string">
  Effective date of the transaction (ISO 8601 format: YYYY-MM-DDTHH:mm:ss). Defaults to created date if not provided.
</ParamField>

### Response

Returns the created accounting entry.

```json
{
  "id": 1003,
  "amount": 2500.00,
  "debitAccountNumber": "1050-001",
  "debitAccountName": "Office Equipment",
  "creditAccountNumber": "1010-001",
  "creditAccountName": "Cash on Hand - Main Branch",
  "description": "Equipment purchase",
  "documentNumber": "EQ-2024-015",
  "branchName": "Head Office",
  "createdByFullName": "John Doe",
  "createdAt": "2024-03-15T09:00:00",
  "effectiveAt": "2024-03-15T09:00:00"
}
```

### Validation Rules

<Warning>
  The following validations are enforced when creating accounting entries:
  
  - Amount must be greater than zero
  - Debit and credit accounts must exist and be different
  - Debit and credit accounts must be BALANCE type (leaf accounts)
  - Accounts must not be locked
  - If `allowedManualTransaction` is false on either account, the entry will be rejected
  - Accounts must have the same currency
  - Description is required and cannot be empty
</Warning>

## Get Account Operations

<CodeGroup>
```bash cURL
curl -X GET "https://your-instance.opencbs.com/api/accounting/accounts/123/operations?from=2024-01-01&to=2024-03-31&page=0&size=20" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://your-instance.opencbs.com/api/accounting/accounts/123/operations?from=2024-01-01&to=2024-03-31&page=0&size=20',
  {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);
const operations = await response.json();
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="number" required>
  Account ID to retrieve operations for
</ParamField>

### Query Parameters

<ParamField query="from" type="string" required>
  Start date (ISO 8601 format: YYYY-MM-DD)
</ParamField>

<ParamField query="to" type="string" required>
  End date (ISO 8601 format: YYYY-MM-DD)
</ParamField>

<ParamField query="page" type="number" default="0">
  Page number
</ParamField>

<ParamField query="size" type="number" default="20">
  Page size
</ParamField>

### Response

Returns a detailed transaction ledger for the account showing deposits, withdrawals, and running balance.

<ResponseField name="content" type="array">
  Array of account operations
  <Expandable title="operation properties">
    <ResponseField name="id" type="number">
      Operation ID
    </ResponseField>
    
    <ResponseField name="date" type="string">
      Transaction date (ISO 8601)
    </ResponseField>
    
    <ResponseField name="deposit" type="number">
      Deposit amount (if applicable)
    </ResponseField>
    
    <ResponseField name="withdraw" type="number">
      Withdrawal amount (if applicable)
    </ResponseField>
    
    <ResponseField name="balance" type="number">
      Running balance after this operation
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Operation description
    </ResponseField>
    
    <ResponseField name="documentNumber" type="string">
      Reference document number
    </ResponseField>
    
    <ResponseField name="source" type="object">
      Source/counterparty account
      <Expandable title="source properties">
        <ResponseField name="id" type="number">Account ID</ResponseField>
        <ResponseField name="number" type="string">Account number</ResponseField>
        <ResponseField name="name" type="string">Account name</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  The operations endpoint provides a ledger view where:
  - **Deposit** represents increases to the account (debits for asset accounts, credits for liability/equity accounts)
  - **Withdraw** represents decreases to the account (credits for asset accounts, debits for liability/equity accounts)
  - **Balance** shows the cumulative balance after each transaction
</Note>

## Common Entry Types

Here are examples of common accounting entries in a microfinance context:

### Cash Deposit

```json
{
  "debitAccountId": 101,  // Cash on Hand
  "creditAccountId": 201, // Member Savings Account
  "amount": 500.00,
  "description": "Member savings deposit",
  "documentNumber": "DEP-2024-123"
}
```

### Loan Disbursement

```json
{
  "debitAccountId": 301,  // Loans Receivable
  "creditAccountId": 101, // Cash on Hand
  "amount": 10000.00,
  "description": "Loan disbursement to member #456",
  "documentNumber": "LOAN-2024-456"
}
```

### Interest Income Recognition

```json
{
  "debitAccountId": 302,  // Interest Receivable
  "creditAccountId": 401, // Interest Income
  "amount": 250.00,
  "description": "Accrued interest on loan #456",
  "documentNumber": "INT-2024-456"
}
```

### Loan Repayment

```json
{
  "debitAccountId": 101,  // Cash on Hand
  "creditAccountId": 301, // Loans Receivable
  "amount": 1000.00,
  "description": "Principal repayment on loan #456",
  "documentNumber": "REPY-2024-456"
}
```

### Expense Payment

```json
{
  "debitAccountId": 501,  // Salary Expense
  "creditAccountId": 101, // Cash on Hand
  "amount": 3500.00,
  "description": "Monthly salaries - March 2024",
  "documentNumber": "SAL-2024-03"
}
```
