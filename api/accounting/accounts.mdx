The Chart of Accounts API provides endpoints for managing your organization's account structure, including creating accounts, browsing the account hierarchy, and configuring account properties.

## Account Hierarchy

OpenCBS uses a hierarchical chart of accounts with four levels:

- **CATEGORY** - Top-level account grouping (Level 1)
- **GROUP** - Second-level grouping (Level 2)
- **SUBGROUP** - Third-level grouping (Level 3)
- **BALANCE** - Leaf accounts that hold actual balances (Level 4)

Only BALANCE accounts can have transactions posted to them.

## Get Root Accounts

<CodeGroup>
```bash cURL
curl -X GET "https://your-instance.opencbs.com/api/accounting/chart-of-accounts/root" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://your-instance.opencbs.com/api/accounting/chart-of-accounts/root',
  {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);
const rootAccounts = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://your-instance.opencbs.com/api/accounting/chart-of-accounts/root',
    headers={'Authorization': 'Bearer YOUR_TOKEN'}
)
root_accounts = response.json()
```
</CodeGroup>

## Create Account

<CodeGroup>
```bash cURL
curl -X POST "https://your-instance.opencbs.com/api/accounting/chart-of-accounts" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "parentAccountId": 10,
    "childNumber": "001",
    "name": "Cash on Hand - Main Branch",
    "currencyId": 1,
    "branchId": 1,
    "isDebit": true,
    "locked": false,
    "allowedTransferFrom": true,
    "allowedTransferTo": true,
    "allowedCashDeposit": true,
    "allowedCashWithdrawal": true,
    "allowedManualTransaction": true
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://your-instance.opencbs.com/api/accounting/chart-of-accounts',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      parentAccountId: 10,
      childNumber: '001',
      name: 'Cash on Hand - Main Branch',
      currencyId: 1,
      branchId: 1,
      isDebit: true,
      locked: false,
      allowedTransferFrom: true,
      allowedTransferTo: true,
      allowedCashDeposit: true,
      allowedCashWithdrawal: true,
      allowedManualTransaction: true
    })
  }
);
const request = await response.json();
```
</CodeGroup>

### Request Body

<ParamField body="parentAccountId" type="number" required>
  ID of the parent account in the hierarchy
</ParamField>

<ParamField body="childNumber" type="string" required>
  Account number suffix (will be appended to parent number)
</ParamField>

<ParamField body="name" type="string" required>
  Account name
</ParamField>

<ParamField body="currencyId" type="number" required>
  Currency ID for this account
</ParamField>

<ParamField body="branchId" type="number" required>
  Branch ID this account belongs to
</ParamField>

<ParamField body="isDebit" type="boolean" required>
  Whether this account has a normal debit balance
</ParamField>

<ParamField body="locked" type="boolean" default="false">
  Lock account from transactions
</ParamField>

<ParamField body="allowedTransferFrom" type="boolean" default="false">
  Allow transfers from this account
</ParamField>

<ParamField body="allowedTransferTo" type="boolean" default="false">
  Allow transfers to this account
</ParamField>

<ParamField body="allowedCashDeposit" type="boolean" default="false">
  Allow cash deposits
</ParamField>

<ParamField body="allowedCashWithdrawal" type="boolean" default="false">
  Allow cash withdrawals
</ParamField>

<ParamField body="allowedManualTransaction" type="boolean" default="false">
  Allow manual journal entries
</ParamField>

### Response

<ResponseField name="id" type="number">
  Request ID for maker-checker approval workflow
</ResponseField>

```json
{
  "id": 456
}
```

<Note>
  Account creation requires maker-checker approval. The returned ID is the request ID, not the account ID. The account will be created after a checker approves the request.
</Note>

## Update Account

<CodeGroup>
```bash cURL
curl -X PUT "https://your-instance.opencbs.com/api/accounting/chart-of-accounts/123" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Cash on Hand - Updated Name",
    "locked": false,
    "allowedTransferFrom": true,
    "allowedTransferTo": true,
    "allowedCashDeposit": true,
    "allowedCashWithdrawal": true,
    "allowedManualTransaction": true
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://your-instance.opencbs.com/api/accounting/chart-of-accounts/123',
  {
    method: 'PUT',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'Cash on Hand - Updated Name',
      locked: false,
      allowedTransferFrom: true,
      allowedTransferTo: true,
      allowedCashDeposit: true,
      allowedCashWithdrawal: true,
      allowedManualTransaction: true
    })
  }
);
const request = await response.json();
```
</CodeGroup>

### Path Parameters

<ParamField path="accountId" type="number" required>
  Account ID to update
</ParamField>

### Request Body

Same structure as create request. Only modifiable fields can be updated.

### Response

```json
{
  "id": 457
}
```

<Note>
  Account updates require maker-checker approval. The account will be updated after a checker approves the request.
</Note>
