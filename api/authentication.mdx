## Overview

OpenCBS Cloud uses JSON Web Tokens (JWT) for API authentication. After successfully logging in, you'll receive a JWT token that must be included in the Authorization header of all subsequent API requests.

The authentication system validates:
- Active user status
- Valid credentials (username and password)
- Password expiration dates
- Session timeouts (configurable)
- First-time login requirements

## Login Endpoint

Authenticate a user and receive a JWT token for API access.

### Request

```http
POST /api/login
```

<ParamField body="username" type="string" required>
  The username or email address of the user
</ParamField>

<ParamField body="password" type="string" required>
  The user's password
</ParamField>

### Response

<ResponseField name="data" type="string">
  JWT token that should be used for subsequent API requests
</ResponseField>

### Example Request

<CodeGroup>

```bash cURL
curl -X POST https://your-instance.opencbs.cloud/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "your-password"
  }'
```

```javascript JavaScript
const response = await fetch('https://your-instance.opencbs.cloud/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    username: 'admin',
    password: 'your-password'
  })
});

const { data } = await response.json();
const token = data;
```

```python Python
import requests

url = 'https://your-instance.opencbs.cloud/api/login'
payload = {
    'username': 'admin',
    'password': 'your-password'
}

response = requests.post(url, json=payload)
token = response.json()['data']
```

```java Java
import com.opencbs.core.dto.requests.LoginRequest;
import com.opencbs.core.dto.responses.ApiResponse;

LoginRequest request = new LoginRequest();
request.setUsername("admin");
request.setPassword("your-password");

// Using RestTemplate or your preferred HTTP client
ApiResponse<String> response = restTemplate.postForObject(
    "https://your-instance.opencbs.cloud/api/login",
    request,
    ApiResponse.class
);

String token = response.getData();
```

</CodeGroup>

## Using the JWT Token

Once you have the JWT token, include it in the `Authorization` header of all API requests using the Bearer authentication scheme.

### Authorization Header Format

```http
Authorization: Bearer <your-jwt-token>
```

### Example Authenticated Request

<CodeGroup>

```bash cURL
curl -X GET https://your-instance.opencbs.cloud/api/users \
  -H "Authorization: Bearer eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJhZG1pbiIsImlzcyI6ImNvbS5vcGVuY2JzLmNvcmUifQ.kxDH8..."
```

```javascript JavaScript
const response = await fetch('https://your-instance.opencbs.cloud/api/users', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
```

```python Python
import requests

headers = {
    'Authorization': f'Bearer {token}',
    'Content-Type': 'application/json'
}

response = requests.get('https://your-instance.opencbs.cloud/api/users', headers=headers)
```

```java Java
HttpHeaders headers = new HttpHeaders();
headers.set("Authorization", "Bearer " + token);
headers.setContentType(MediaType.APPLICATION_JSON);

HttpEntity<String> entity = new HttpEntity<>(headers);
ResponseEntity<String> response = restTemplate.exchange(
    "https://your-instance.opencbs.cloud/api/users",
    HttpMethod.GET,
    entity,
    String.class
);
```

</CodeGroup>

## JWT Token Details

The JWT tokens generated by OpenCBS Cloud have the following characteristics:

<ResponseField name="Algorithm" type="HS512">
  Tokens are signed using HMAC with SHA-512
</ResponseField>

<ResponseField name="Subject" type="string">
  Contains the username of the authenticated user
</ResponseField>

<ResponseField name="Issuer" type="string">
  Always set to `com.opencbs.core`
</ResponseField>

<ResponseField name="Expiration" type="configurable">
  Token lifetime is controlled by the `EXPIRATION_SESSION_TIME_IN_MINUTES` system setting. Set to 0 for tokens that never expire.
</ResponseField>

### Token Validation

The API validates tokens on each request by:

1. Extracting the username from the JWT token
2. Verifying the user exists and has `ACTIVE` status
3. Checking the token signature is valid
4. Validating the session hasn't expired based on the user's last activity time

<Warning>
  Session expiration is tracked based on the user's last entry time. Each authenticated request updates this timestamp, keeping the session alive.
</Warning>

## Logout

Invalidate a user's session.

### Request

```http
POST /api/logout/{userId}
```

<ParamField path="userId" type="integer" required>
  The ID of the user to logout
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl -X POST https://your-instance.opencbs.cloud/api/logout/123 \
  -H "Authorization: Bearer eyJhbGciOiJIUzUxMiJ9..."
```

```javascript JavaScript
await fetch(`https://your-instance.opencbs.cloud/api/logout/${userId}`, {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

```python Python
import requests

headers = {'Authorization': f'Bearer {token}'}
requests.post(f'https://your-instance.opencbs.cloud/api/logout/{user_id}', headers=headers)
```

</CodeGroup>

## Password Management

### Update Password

Change a user's password.

#### Request

```http
PUT /api/login/update-password
```

<ParamField body="userId" type="integer" required>
  The ID of the user whose password should be updated
</ParamField>

<ParamField body="password" type="string" required>
  The new password (must meet password complexity requirements)
</ParamField>

#### Example Request

<CodeGroup>

```bash cURL
curl -X PUT https://your-instance.opencbs.cloud/api/login/update-password \
  -H "Authorization: Bearer eyJhbGciOiJIUzUxMiJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "userId": 123,
    "password": "NewSecurePassword123!"
  }'
```

```javascript JavaScript
await fetch('https://your-instance.opencbs.cloud/api/login/update-password', {
  method: 'PUT',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    userId: 123,
    password: 'NewSecurePassword123!'
  })
});
```

```python Python
import requests

headers = {
    'Authorization': f'Bearer {token}',
    'Content-Type': 'application/json'
}

payload = {
    'userId': 123,
    'password': 'NewSecurePassword123!'
}

requests.put('https://your-instance.opencbs.cloud/api/login/update-password', 
             headers=headers, 
             json=payload)
```

</CodeGroup>

#### Example Response

```text
Password has been successfully changed
```

### Password Reset

Request a password reset for a user. A randomly generated password will be sent to the user's email address.

#### Request

```http
POST /api/login/password-reset?username={username}
```

<ParamField query="username" type="string" required>
  The username or email address of the user requesting password reset
</ParamField>

#### Example Request

<CodeGroup>

```bash cURL
curl -X POST "https://your-instance.opencbs.cloud/api/login/password-reset?username=admin" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const username = 'admin';
await fetch(`https://your-instance.opencbs.cloud/api/login/password-reset?username=${username}`, {
  method: 'POST'
});
```

```python Python
import requests

username = 'admin'
requests.post(f'https://your-instance.opencbs.cloud/api/login/password-reset?username={username}')
```

</CodeGroup>

#### Example Response

```text
Check your email.
```

<Warning>
  The user must have a valid email address configured to receive password reset emails. The system generates a 15-character random password containing letters and numbers.
</Warning>

## Error Responses

### Invalid Credentials

When the username or password is incorrect:

```json
{
  "status": 404,
  "error": "Not Found",
  "message": "Incorrect username or password."
}
```

### User Disabled

When attempting to login with a disabled user account:

```json
{
  "status": 401,
  "error": "Unauthorized",
  "message": "User is disabled.",
  "userId": "123"
}
```

### Password Expired

When the user's password has expired:

```json
{
  "status": 401,
  "error": "Unauthorized",
  "message": "Password expired",
  "userId": "123"
}
```

### First Login Required

When first-time password change is required:

```json
{
  "status": 401,
  "error": "Unauthorized",
  "message": "Set a new password",
  "userId": "123"
}
```

### Invalid or Missing Token

When the JWT token is missing, invalid, or expired:

```json
{
  "status": 401,
  "error": "Unauthorized",
  "message": "Authentication required"
}
```

## Best Practices

### Secure Token Storage

- **Client-side applications**: Store tokens in memory or secure storage (never localStorage in browsers)
- **Server-side applications**: Store tokens securely with encryption at rest
- **Mobile applications**: Use platform-specific secure storage (Keychain on iOS, KeyStore on Android)

### Token Lifecycle Management

1. **Request a token**: Call `/api/login` with valid credentials
2. **Store securely**: Keep the token in secure storage
3. **Include in requests**: Add to Authorization header for all API calls
4. **Handle expiration**: Monitor session timeout and re-authenticate when needed
5. **Logout**: Call `/api/logout/{userId}` when the user ends their session

### Session Management

- Configure `EXPIRATION_SESSION_TIME_IN_MINUTES` appropriately for your security requirements
- Set to `0` for long-lived tokens (use with caution)
- Shorter timeouts (15-60 minutes) improve security but may impact user experience
- Each API request refreshes the session timestamp, extending the expiration time

### Error Handling

Always check for authentication errors and handle them appropriately:

```javascript
const response = await fetch(url, {
  headers: { 'Authorization': `Bearer ${token}` }
});

if (response.status === 401) {
  // Token expired or invalid - redirect to login
  redirectToLogin();
}
```

<Warning>
  Never include JWT tokens in URL query parameters or log files. Always use the Authorization header for token transmission.
</Warning>

## Security Considerations

1. **HTTPS Required**: Always use HTTPS in production to prevent token interception
2. **Password Policies**: Ensure strong password validation is configured via `UserDtoValidator`
3. **Account Status**: Only users with `ACTIVE` status can authenticate
4. **Session Expiration**: Configure appropriate timeout values based on your security needs
5. **Logout Implementation**: Always provide a way for users to explicitly end their session
6. **Token Rotation**: Consider implementing token refresh mechanisms for long-running applications
