## Overview

Savings products define the templates and rules for savings accounts. They specify interest rates, fees, transaction limits, and other parameters that govern how savings accounts operate.

All product operations follow a **maker-checker approval workflow** to ensure proper controls on configuration changes.

## List Savings Products

<CodeGroup>
```bash cURL
curl -X GET "https://api.opencbs.com/api/saving-products?search=basic&show_all=false&page=0&size=20" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/saving-products?page=0&size=20', {
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});
const products = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.opencbs.com/api/saving-products',
    params={'page': 0, 'size': 20},
    headers={'Authorization': 'Bearer YOUR_ACCESS_TOKEN'}
)
products = response.json()
```
</CodeGroup>

## Create Savings Product

Create a new savings product. This operation creates a maker-checker request that requires approval.

<Note>
  Requires `MAKER_FOR_SAVING_PRODUCT` permission. Returns a request ID that must be approved by a checker before the product becomes active.
</Note>

<CodeGroup>
```bash cURL
curl -X POST "https://api.opencbs.com/api/saving-products" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Premium Savings",
    "code": "SAV-002",
    "currencyId": 1,
    "availability": ["PERSON"],
    "initialAmountMin": 500.00,
    "initialAmountMax": 50000.00,
    "interestRateMin": 3.5,
    "interestRateMax": 6.0,
    "interestAccrualFrequency": "DAILY",
    "postingFrequency": "MONTHLY",
    "capitalized": true,
    "depositAmountMin": 50.00,
    "depositAmountMax": 100000.00,
    "withdrawalAmountMin": 50.00,
    "withdrawalAmountMax": 50000.00,
    "minBalance": 100.00,
    "accounts": {
      "savingAccountId": 1100,
      "interestAccrualAccountId": 1110,
      "interestExpenseAccountId": 5100
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/saving-products', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Premium Savings',
    code: 'SAV-002',
    currencyId: 1,
    availability: ['PERSON'],
    initialAmountMin: 500.00,
    initialAmountMax: 50000.00,
    interestRateMin: 3.5,
    interestRateMax: 6.0,
    interestAccrualFrequency: 'DAILY',
    postingFrequency: 'MONTHLY',
    capitalized: true,
    depositAmountMin: 50.00,
    depositAmountMax: 100000.00,
    withdrawalAmountMin: 50.00,
    withdrawalAmountMax: 50000.00,
    minBalance: 100.00

});
const request = await response.json();
```
</CodeGroup>

## Update Savings Product

Update an existing savings product. This operation creates a maker-checker request that requires approval.

<Warning>
  Products that are in use by active accounts may have restrictions on which fields can be modified.
</Warning>

<CodeGroup>
```bash cURL
curl -X PUT "https://api.opencbs.com/api/saving-products/1" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Basic Savings Plus",
    "interestRateMin": 2.5,
    "interestRateMax": 5.5,
    "depositAmountMax": 75000.00
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/saving-products/1', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Basic Savings Plus',
    interestRateMin: 2.5,
    interestRateMax: 5.5,
    depositAmountMax: 75000.00
  })
});
const request = await response.json();
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="long" required>
  Savings product ID to update
</ParamField>

### Request Body

Accepts the same fields as Create Savings Product. Only include fields you want to update.

### Response

Returns a `RequestDto` object containing the maker-checker request ID.

```json
{
  "id": 457
}
```

```bash cURL
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.opencbs.com/api/saving-products/1/history/last_change?dateTime=2024-01-10T12:00:00',
  {
    headers: {
      'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
    }

const historicalProduct = await response.json();
```
</CodeGroup>
