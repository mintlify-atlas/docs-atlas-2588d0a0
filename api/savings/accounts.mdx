## Overview

Savings accounts represent individual customer savings relationships. Each account is based on a savings product template and linked to a customer profile (person or company).

## List Savings Accounts

Retrieve a paginated list of all savings accounts with search capability.

<Note>
  Requires `GET_SAVINGS` permission.
</Note>

<CodeGroup>
```bash cURL
curl -X GET "https://api.opencbs.com/api/savings?search=john&page=0&size=20" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/savings?search=john&page=0&size=20', {
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});
const accounts = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.opencbs.com/api/savings',
    params={'search': 'john', 'page': 0, 'size': 20},
    headers={'Authorization': 'Bearer YOUR_ACCESS_TOKEN'}
)
accounts = response.json()
```
</CodeGroup>

### Query Parameters

<ParamField query="search" type="string">
  Search term to filter accounts by account code, customer name, or other attributes
</ParamField>

<ParamField query="page" type="integer" default="0">
  Page number for pagination (0-indexed)
</ParamField>

<ParamField query="size" type="integer" default="20">
  Number of items per page
</ParamField>

<ParamField query="sort" type="string">
  Sort field and direction (e.g., `openDate,desc` or `code,asc`)
</ParamField>

### Response

Returns a paginated list of `SavingSimplified` objects.


## Create Savings Account

Create a new savings account in PENDING status. The account must be opened separately using the Open endpoint.

<Note>
  Requires `SAVINGS_CREATE` permission.
</Note>

<CodeGroup>
```bash cURL
curl -X POST "https://api.opencbs.com/api/savings" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "profileId": 45,
    "savingProductId": 1,
    "savingOfficerId": 10,
    "interestRate": 3.5,
    "openDate": "2024-01-15T09:00:00",
    "depositFeeRate": 0.0,
    "depositFeeFlat": 2.0,
    "withdrawalFeeRate": 1.0,
    "withdrawalFeeFlat": 5.0,
    "entryFeeRate": 0.0,
    "entryFeeFlat": 10.0,
    "closeFeeRate": 0.0,
    "closeFeeFlat": 25.0,
    "managementFeeRate": 0.0,
    "managementFeeFlat": 5.0
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/savings', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    profileId: 45,
    savingProductId: 1,
    savingOfficerId: 10,
    interestRate: 3.5,
    openDate: '2024-01-15T09:00:00',
    depositFeeFlat: 2.0,
    withdrawalFeeRate: 1.0,
    withdrawalFeeFlat: 5.0
  })
});
const account = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.opencbs.com/api/savings',
    json={
        'profileId': 45,
        'savingProductId': 1,
        'savingOfficerId': 10,
        'interestRate': 3.5,
        'openDate': '2024-01-15T09:00:00',
        'depositFeeFlat': 2.0,
        'withdrawalFeeRate': 1.0,
        'withdrawalFeeFlat': 5.0
    },
    headers={'Authorization': 'Bearer YOUR_ACCESS_TOKEN'}
)
account = response.json()
```
</CodeGroup>

### Request Body

<ParamField body="profileId" type="long" required>
  Customer profile ID (must be a valid PERSON or COMPANY profile)
</ParamField>

<ParamField body="savingProductId" type="long" required>
  Savings product ID to use as template
</ParamField>

<ParamField body="savingOfficerId" type="long" required>
  User ID of the savings officer managing this account
</ParamField>

<ParamField body="interestRate" type="decimal" required>
  Annual interest rate percentage (must be within product's min/max range)
</ParamField>

<ParamField body="openDate" type="string">
  Planned opening date and time (ISO 8601 format: `yyyy-MM-dd'T'HH:mm:ss`)
</ParamField>

<ParamField body="depositFeeRate" type="decimal">
  Deposit fee as percentage (must be within product range)
</ParamField>

<ParamField body="depositFeeFlat" type="decimal">
  Flat deposit fee amount (must be within product range)
</ParamField>

<ParamField body="withdrawalFeeRate" type="decimal">
  Withdrawal fee as percentage (must be within product range)
</ParamField>

<ParamField body="withdrawalFeeFlat" type="decimal">
  Flat withdrawal fee amount (must be within product range)
</ParamField>

<ParamField body="entryFeeRate" type="decimal">
  Opening fee as percentage of initial deposit
</ParamField>

<ParamField body="entryFeeFlat" type="decimal">
  Flat opening fee amount
</ParamField>

<ParamField body="closeFeeRate" type="decimal">
  Closing fee as percentage of balance
</ParamField>

<ParamField body="closeFeeFlat" type="decimal">
  Flat closing fee amount
</ParamField>

<ParamField body="managementFeeRate" type="decimal">
  Management fee as percentage
</ParamField>

<ParamField body="managementFeeFlat" type="decimal">
  Flat management fee amount
</ParamField>

### Response

Returns the created `SavingDetailsDto` object with status PENDING.

```json
{
  "id": 103,
  "code": "SAV-00103",
  "profileId": 45,
  "profileName": "John Doe",
  "profileType": "PERSON",
  "savingProductId": 1,
  "savingProductName": "Basic Savings",
  "savingOfficerId": 10,
  "savingOfficerName": "Sarah Johnson",
  "status": "PENDING",
  "interestRate": 3.5,
  "openDate": "2024-01-15T09:00:00",
  "savingBalance": 0.00,
  "accruedInterest": 0.00,
  "locked": false
}
```

## Update Savings Account

Update an existing savings account's configuration. Only PENDING accounts can have all fields updated. ACTIVE accounts have restrictions.

<Note>
  Requires `SAVINGS_UPDATE` permission.
</Note>

<CodeGroup>
```bash cURL
curl -X PUT "https://api.opencbs.com/api/savings/103" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "interestRate": 4.0,
    "withdrawalFeeFlat": 7.5
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opencbs.com/api/savings/103', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    interestRate: 4.0,
    withdrawalFeeFlat: 7.5
  })
});
const account = await response.json();
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="long" required>
  Savings account ID
</ParamField>

### Request Body

Accepts the same fields as Create Savings Account. Only include fields you want to update.

### Response

Returns the updated `SavingDetailsDto` object.

## Open Savings Account

Open a PENDING savings account by making an initial deposit. This transitions the account to ACTIVE status.

<Note>
  Requires `SAVING_OPEN` permission.
</Note>

<CodeGroup>
```bash cURL
curl -X POST "https://api.opencbs.com/api/savings/103/open?initialAmount=500.00" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.opencbs.com/api/savings/103/open?initialAmount=500.00',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
    }
  }
);
const account = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.opencbs.com/api/savings/103/open',
    params={'initialAmount': 500.00},
    headers={'Authorization': 'Bearer YOUR_ACCESS_TOKEN'}
)
account = response.json()
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="long" required>
  Savings account ID
</ParamField>

### Query Parameters

<ParamField query="initialAmount" type="decimal" required>
  Initial deposit amount (must be within product's min/max range)
</ParamField>

### Response

Returns the updated `SavingDetailsDto` object with status ACTIVE and the initial balance.

```json
{
  "id": 103,
  "code": "SAV-00103",
  "status": "ACTIVE",
  "savingBalance": 490.00,
  "openDate": "2024-01-15T09:00:00",
  "accruedInterest": 0.00
}
```

<Note>
  Entry fees (if configured) are automatically deducted from the initial deposit. In the example above, a $10 entry fee was deducted from the $500 initial deposit.
</Note>

## Close Savings Account

Close an ACTIVE savings account. The remaining balance is withdrawn and the account is set to CLOSED status.

<Warning>
  This operation is irreversible. Ensure the account is fully actualized (interest calculated) before closing.
</Warning>

<Note>
  Requires `SAVING_CLOSE` permission.
</Note>

<CodeGroup>
```bash cURL
curl -X POST "https://api.opencbs.com/api/savings/103/close?closeDate=2024-06-30" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.opencbs.com/api/savings/103/close?closeDate=2024-06-30',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
    }
  }
);
const account = await response.json();
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="long" required>
  Savings account ID
</ParamField>

### Query Parameters

<ParamField query="closeDate" type="string" required>
  Closing date in ISO 8601 date format (`yyyy-MM-dd`). Must be on or after the account opening date.
</ParamField>

### Response

Returns the updated `SavingDetailsDto` object with status CLOSED and zero balance.

```json
{
  "id": 103,
  "code": "SAV-00103",
  "status": "CLOSED",
  "savingBalance": 0.00,
  "accruedInterest": 0.00,
  "openDate": "2024-01-15T09:00:00"
}
```

<Note>
  Closing fees (if configured) are automatically deducted from the balance before the final withdrawal.
</Note>

## Actualize Savings Account

Manually trigger the interest accrual process for a specific savings account up to a given date.

<Note>
  Requires `ACTUALIZE_SAVING` permission. This process calculates and posts any pending interest.
</Note>

<CodeGroup>
```bash cURL
curl -X POST "https://api.opencbs.com/api/savings/actualize/103?date=2024-01-31" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.opencbs.com/api/savings/actualize/103?date=2024-01-31',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
    }
  }
);
const message = await response.text();
```
</CodeGroup>

### Path Parameters

<ParamField path="savingId" type="long" required>
  Savings account ID
</ParamField>

### Query Parameters

<ParamField query="date" type="string" required>
  Date to actualize up to (ISO 8601 date format: `yyyy-MM-dd`)
</ParamField>

### Response

Returns a success message.

```
Actualize saving started
```

<Note>
  Actualization runs asynchronously. The account balance and accrued interest are updated once the process completes.
</Note>
